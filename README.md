# CourseWeaver AI Studio

**CourseWeaver AI** — это клиентское SPA (Single Page Application) для автоматизированного проектирования и создания образовательных онлайн-курсов. Приложение специализируется на формате LMS Stepik, генерируя структуру (модули, уроки) и детализированный контент (тексты, сценарии видео, тесты) с использованием новейших моделей Google Gemini (3.0 Pro, 2.5 Flash).

---

## 🛠 Технический Стек

*   **Framework:** React 19
*   **Language:** TypeScript
*   **Styling:** Tailwind CSS + Typography plugin (Dark/Light mode support)
*   **AI SDK:** `@google/genai` (прямое взаимодействие с API Gemini)
*   **Build/Runtime:** Browser-native ES Modules (через index.html importmap)
*   **Persistence:** LocalStorage (история курсов, настройки темы и модели)

## 🧠 Архитектура и Логика работы

Приложение построено по монолитной архитектуре клиентской части. Бэкенд отсутствует; все операции по генерации, сохранению и обработке данных происходят в браузере пользователя.

### 1. Поток данных (Data Flow)

1.  **Сбор контекста (Input Phase):**
    *   Пользователь вводит тему, ЦА, выбирает тональность.
    *   Загружаются файлы контекста (.txt, .md). Они объединяются в единую строку промпта.
    *   *Компонент:* `App.tsx` (RenderInput).

2.  **Генерация концепции (Concept Phase):**
    *   Вызывается `generateCourseConcept`.
    *   **LLM Logic:** Модель получает роль методиста Stepik и возвращает JSON с метаданными (описание, чему научатся, программа).
    *   Используется механизм `extractAndParseJSON` для надежного извлечения JSON из ответов LLM (с балансировкой скобок и очисткой Markdown).

3.  **Детализация структуры (Structure Phase):**
    *   Вызывается `generateDetailedCourseStructure`.
    *   Текстовая программа (Syllabus) преобразуется в иерархический массив объектов: `Module[] -> Lesson[] -> LessonBlock[]`.
    *   Пользователь может редактировать структуру вручную (Drag & Drop, удаление/добавление).

4.  **Генерация контента (Content Phase):**
    *   **Гранулярная генерация:** Можно сгенерировать контент для конкретного урока (`generateFullLessonContent`) или отдельного блока.
    *   **Пакетная генерация:** Функция `handleGenerateAllCourseContent` итерируется по всем урокам и последовательно запрашивает генерацию контента, обновляя UI в реальном времени.
    *   **Типы блоков:** Поддерживаются специфичные для Stepik типы: `text`, `video` (сценарий), `quiz_choice`, `matching`, `sorting`, `open_answer`.

5.  **Экспорт:**
    *   Генерация Markdown или HTML файлов через `services/exportService.ts`.
    *   Структура Markdown оптимизирована для импорта в базы знаний (Obsidian/Notion).

### 2. Ключевые Сервисы (`services/geminiService.ts`)

Это ядро приложения. Содержит функции-обертки над `@google/genai`:

*   **`generateCourseConcept`**: Создает "скелет" курса. Использует большие контекстные окна (до 65k токенов вывода) для обработки загруженных книг/методичек.
*   **`generateLessonsForModule` / `generateBlocksForLesson`**: Функции для "точечного" расширения структуры, когда пользователь добавляет пустые модули вручную.
*   **`refineSection`**: Режим "ИИ-редактора". Позволяет выделить любой текстовый блок и попросить ИИ переписать его (например, "сделай проще", "добавь пример").
*   **`generateCourseCoverImage`**: Использует модель `gemini-2.5-flash-image` для генерации обложки курса по теме.

### 3. Структура Данных (`types.ts`)

Приложение оперирует сложным вложенным объектом `CourseConcept`:

```typescript
interface CourseConcept {
  title: string;
  syllabus: string; // HTML строка программы
  modules: Module[]; // Детальная структура
  // ...метаполя Stepik (workload, level, etc.)
}

interface Module {
  lessons: Lesson[];
}

interface Lesson {
  blocks: LessonBlock[];
}

interface LessonBlock {
  type: 'text' | 'video' | 'quiz_choice' | ...;
  contentSummary: string; // Промпт/Описание для генерации
  fullContent?: string;   // Сгенерированный HTML контент
}
```

## 🧩 Компонентная База

*   **`App.tsx`**: Основной контроллер. Управляет состоянием (Step, Concept, History, Modals). Содержит логику Drag&Drop перемещения блоков и уроков. Реализует UI бокового меню и навигации.
*   **`EditableSection.tsx`**: Универсальный компонент для отображения и редактирования контента. Имеет три режима:
    1.  **View:** Рендеринг HTML (с защитой стилей).
    2.  **Manual:** Редактирование сырого текста/HTML.
    3.  **AI:** Поле ввода промпта для регенерации секции.
*   **`CourseCard.tsx` / `Button.tsx`**: UI примитивы с поддержкой тем (Tailwind).

## 🚀 Особенности реализации

1.  **Устойчивость к ошибкам JSON:**
    LLM часто возвращают JSON с ошибками или внутри Markdown блоков. Функция `extractAndParseJSON` реализует 3 стратегии парсинга (чистый, regex extraction, балансировка скобок), что минимизирует сбои генерации.

2.  **Интерактивный редактор структуры:**
    В отличие от линейных генераторов, CourseWeaver позволяет вмешиваться в процесс на любом этапе:
    *   Вставить модуль посередине -> Сгенерировать для него уроки.
    *   Вставить урок -> Сгенерировать для него блоки.
    *   Вставить блок -> Сгенерировать для него контент.

3.  **Темная тема и UI:**
    Используется `class="dark"` на теге `html`. Tailwind классы настроены через модификатор `dark:`. Для типографики используется плагин `@tailwindcss/typography` с кастомными оверрайдами цветов для темного режима.

4.  **Интеграция с Google Search (Grounding):**
    В коде заложена (но опциональна через UI) возможность использования `tools: [{ googleSearch: {} }]` для актуализации информации в курсах.

## ⚠️ Ограничения и Известные проблемы

1.  **Контекстное окно:** При очень больших загруженных файлах (>1 млн токенов) браузер может подвиснуть при отправке запроса, так как файл читается в память целиком.
2.  **Local Storage:** История хранится в локальном хранилище браузера. Очистка кэша удалит все курсы.
3.  **Rate Limits:** При нажатии "Сгенерировать ВЕСЬ курс" происходит последовательный перебор уроков. При большом курсе можно упереться в лимиты API Gemini (RPM/TPM).
4.  **Безопасность:** API Key используется на клиенте (`process.env.API_KEY`). Это допустимо для локального инструмента/прототипа, но требует прокси-сервера для продакшена.

## 📦 Установка и Запуск

Приложение не требует сборки (No-Build) в традиционном понимании Node.js, так как использует ESM импорты через CDN, но для удобства разработки оно структурировано как React-проект.

1.  Убедитесь, что у вас есть API Key от Google Gemini.
2.  Ключ должен быть доступен в `process.env.API_KEY`.
3.  Откройте приложение в среде, поддерживающей React 19 (например, локальный сервер Vite или StackBlitz).
